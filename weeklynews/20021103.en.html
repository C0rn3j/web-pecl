PEAR-DEV traffic

Payment_Bibit back to Life

Dave Mertens, who began a class to work with Bibit payment, has been swamped with requests since he removed the incomplete class from PEAR, he posted that he was going to try and attempt to finish it in his spare time, as there was so much demand.

go-pear goes web based

Christian Dickmann announce a trial version of the Web based pear installer, that could be downloaded and installed, helping to bootstrap the pear package manager in cases where command line access was difficult, you can help test it out by downloading it from

http://dickmann.homeunix.org/pear/pearweb/public_html/go-pear.php

PEAR's mention at PHPCon (US), as Yahoo moves to PHP

PEAR got a breif mention at PHPCon in the US, during the reasoning behind Yahoo beginning migration of alot of it's sites to PHP. While it mentioned that PEAR does not yet compare to CPAN, all regular readers know that the aim (when PHP4.3) is release is to provide something that is better (in quality, if not quantity yet) than CPAN.

Java DataObjects equivialent in PEAR

Andi Guntmans (of Zend Fame), posted asking about an equivilant to Java DataObjects in PEAR. While DB_DataObjects provides some of the features he was asking, he was keen to see a more complete implementation, matching and exceeding the functionality of the Java Version. Watch this space for more developments.

New and Proposed Packages

Bytecode Optimizer

George Schlossnagle announced his intention to commit a new tool to optimize PHP bytecodes, this would be of benifit to APC and bcompiler, and would provide an open source alternative to the optimizers provided by Zend and InCube.

Genealogy_Gedcom - A specialized class for Genealogy

Olivier Vanhoucke announced a class which helps with Geneology analysis, for those working in this field, obviously of great use.
