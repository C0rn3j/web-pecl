Outra semana quieta, já que alguns desenvolvedores do PEAR estão se preparando para a PHP Conference em Frankfurt (venha ver quem está por trás de todo aquele código). Discussões sobre o dirname(__FILE__) continuaram, com o "arquiteto" original explicando as razões para tal decisão. Três novos pacotes foram propostos esta semana, já que o PEAR continua a atrair o código e extensões PHP da melhor qualidade, além de uma quantidade contínua de novos lançamentos: 4 estáveis e 4 betas.

Novos Lançamentos

Estável

  • HTML_Select - provê caixas de seleção para países, países do Reino Unido e estados dos EUA
  • Log - um framework para prover log
  • HTML_QuickForm - um pacote que provê métodos para criar, validar e processar formulários HTML
  • HTML_Table - métodos para o desenho fácil e eficiente de tabelas HTML

Beta

  • PEAR_Frontend_Web - a interface web para o instalador do PEAR
  • SOAP_Interop - um conjunto de testes para serviços SOAP
  • Net_DIME - um pacote que provê uma interface para o DIME (protocolo de mensagens instantâneas)
  • SOAP - um conjunto de bibliotecas que permitem ao PHP atuar como servidor e cliente SOAP

Tráfego na Lista PEAR-DEV

Continuação - Paths do PEAR no require_once e o uso do dirname(__FILE__)

Seguindo a discussão das últimas semanas em usar dirname(__FILE__) para os "caminhos", Stig Bakken indicou os sérios problema de usar isto em conjunto com múltiplas instalações do PEAR (global + usuário). O problema está no fato de que o require_once poderia se confundir seriamente e pode terminar carregando duas classes com o mesmo nome em dois diferentes arquivos, causando um conflito. O método atual de deixar o include path fazer tal classificação parece ser a única solução viável atualmente.

Novos containers MDB para o PEAR cache e mail queue

Lorenzo Alberton anunciou que escreveu containers do MDB para o PEAR cache e o mail queue.

Notas do usuário para o manual do PEAR

Foi sugerido por Jon Wood que um dos mais populares características do manual do PHP fosse adicionado ao manual do PEAR. Não houveram muitas respostas. Tudo indica que quando o PEARDoc2 estiver no ar esta sera uma funcionalidade importante que será adicionada.

Agradecimentos a estas pessoas pela detecção e correçao de alguns erros: David Viner, Daniel C, Lorenzo Alberton, Sebastian Röbke, Mika Tuupola, Alan Richmond e Robin Ericsson

Pacotes Novos e Propostos

APC (Cache Avançado do PHP) agora em PECL

George Schlossnagle anunciou sua intenção de adicionar o APC, um cache PHP disponível livremente, ao PECL.

ezSQL - uma API a banco de dados simplificada

Justin Vncent propôs sua API para banco de dados simplificada para uso do PEAR, depois de algumas confusões sobre como ele se encaixaria no PEAR. Um concenso geral é de que este tipo de wrapper como uma extensão ao DB ou MDB pode ser útil para alguns usuários. Apesar de que os desenvolvedores do PEAR "puristas" não concordaram tão firmemente com o encapsulamento dos detalhes de conexão ao banco de dados.

Net_Cyrus - uma interface para os comandos de gerenciamento do servidor IMAP Cyrus

Richard Heyes propôs uma pacote para fornecer uma interface ao servidor de mails IMAP Cyrus, permitindo a adição de novos usuários e alterações de seus direitos de acesso.
