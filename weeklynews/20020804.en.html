PEAR-DEV traffic

Ready made binaries and Package Signing

Following some discussion on IRC,  Marko Karppinen Posted the plans for distributing binaires of PECL (php extensions). Following the recent OpenSSH incident where somebody put malicous code in a source distribution, there is obviously a heightend need for additional security and checks here. For more details on the proposal please read the post here

Making Documentation easier

Currently underway and under consideration are 3 projects to try and make writing documentation PEAR easier.
  • HTML to Docbook, Martin Jansen proposed a very simple mapping of HTML type <H1> type tags to docbook - see the notes here
  • Phpdoc to Docbook, Alexander and Alan both mentioned this - Alans approach was to extend the PHP_Codedoc tool that he had written to do this.
  • OpenOffice to Docbook,  Wolfram Kriesing was considering writing a convertor for openoffice XML -> docbook

Package Statistics Improved

Tal Peer, added a full summary for the package statistics, so you can see a list of how many times all the packages has been downloaded. 

High Speed file access.

After a brief dicussion about performance and file reading, Thomas Cox posted a  quick benchmark  illustrating fread() being noticably faster than implode('',file())

new PECL extenstion - PICOSQL

Bindings for the picosql database have been added to PECL, picosql is a proprietary relational database that has 'brought out into the open', with a LGPL licence.

new PECL extenstion - python

John Praise Added his python extension to PECL, enabling you to mix Python and PHP, and access Python variables from php.

PHPDoc back to life

The Award Winning Derick Rethans has picked up the gauntlet of fixing up PHPdoc, and is looking for suggestions about what could be added to it.
