A brief digest: the dominating topic was LiveUser, Stig registered new PEAR related domain names and PEAR&privacy.

New Releases

Stable

  • XML_Transformer - transforms XML tags to PHP code or text (similar concept to XSLT)
  • XML_RPC - an API for handling Remote Procedure Calls
  • PEAR - the PEAR base package

Beta

  • Translation - an API to handle I18N of string resources
  • Config - an API to process and editing configuration datasources

Alpha

  • LiveUser - user authentication and permission management
  • XML_XSLT_Wrapper - abstraction layer for XSL transformation

PEAR-DEV traffic

Category Perm?

Markus Wolff asked for a new category called Perm for LiveUser. There was no agreement and Markus released LiveUser under the category Authentication.

Go PEAR, go!

Stig Bakken registered the web domains go-pear.org|.com|.net. They serve the go-pear script, which installs a current working PEAR base system including the PEAR installer. If you want a working PEAR on your system just run on your prompt:

lynx -source go-pear.org|php

PEAR&Privacy

Derick Rethans noted some possible privacy problems on the PEAR web. If you download a package, your IP will be stored for statistical reasons. This could hurd laws in some european countries. There was no decision how to handle this - we are programmer, not lawyers. If you more know then we, then wrote a note to the PEAR-Dev mailing list.

New and Proposed Packages

HTML_Select_Country

Richard Heyes and Marten Stolte posted some ideas about a Package for creating an pulldown menu for choosing an country name. This includes localized lists of the names.
