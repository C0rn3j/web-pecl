Après quelques semaines fiévreuses, la semaine écoulée à été plutôt calme avec deux livraisons stables et deux beta. Des discussions ont pris place sur divers sujets: rendre l'écriture de documentation plus facile, de nouvelles statistiques sur les packages, les livraisons d'éxecutables signés. PhpDoc est revenu à la vie et deux nouvelles extensions PECL ont vu le jour.

Packages éxecutables et signés

Après discussion sur IRC, Marko Karppinen à posté le processus de distribution des éxecutables PECL (extensions PHP). Après l'incident ayant vu un utilisateur mettre du code malicieux dans une distribution d'OpenSSH il est évident qu'il est nécessaire d'améliorer la sécurité et les vérifications. Pour plus de détails sur cette proposition veuillez lire le fil de discussion ici (ndt: en anglais)

Rendre l'effort de documentation plus facile

Trois projets, en cours ou en analyse, pour rendre l'écriture de documentation pour PEAR plus simple.
  • HTML vers Docbook, Martin Jansen à proposé une simple conversion de tags de type HTML vers un format DocBook - veuillez lire le message ici (ndt: en anglais)
  • Phpdoc vers Docbook, Alexander and Alan ont deux mentionnés qu'une des solution est d'étendre l'outil PHP_Codedoc d'Alan pour effectuer cette conversion. Alexander va écrire un convertisseur pour transformer des commentaires PhpDoc, le résultat devrait pouvoir être inséré au manuel de PEAR.
  • OpenOffice vers Docbook,  Wolfram Kriesing étudie la possibilité d'écrire un convertisseur pour passer du format OpenOffice XML vers Docbook

Amélioration des statistiques sur les packages

Tal Peer à ajouté un résumé complet (ndt: en anglais) sur les statistiques des packages. Vous pouvez ainsi savoir combien de fois un package à été téléchargé.

Accès fichier rapide

Après une brève discussion Tomas V. Cox à posté a un rapide banc d'essai montrant la plus grande rapidité de fread() comparé à implode('',file()).

PHPDoc revient à la vie

Le gagnant d'un oscar Derick Rethans a relevé ses manches pour s'attaquer à corriger PhpDoc. Il est ouvert à toutes suggestions sur les fonctionnalités à y ajouter.

Propositions et nouveaux packages

PICOSQL

Une librairie de liaison vers le SGBD picosql à été ajoutée à PECL. PICOSQL est un système de gestion de base de données relationnelle propriétaire qui à été ouvert à tous grâce au passage à la licence LGPL.

Python

John Praise a jouté une extension Python à PECL qui permettant de mélanger Python et PHP,  ainsi qu'accéder aux variables Python depuis PHP.
